Введение в Document Object Model (DOM) и его свойства

Основы DOM:
DOM API позволяет взаимодействовать с HTML-структурой страницы: выбирать элементы, изменять их, добавлять новые или удалять существующие.
DOM представляет собой объектное представление HTML-документа, с которым JavaScript может взаимодействовать.
DOM служит соединителем между HTML и JavaScript, обеспечивая доступность HTML-элементов для скриптов.
Структура DOM:
Все элементы в DOM документе представляются в виде узлов (nodes) в дереве.
Каждый узел может быть элементом, текстом, комментарием и относится к одному из подклассов класса Node.
Наследование и доступ к элементам:
В основе иерархии DOM лежит класс Node, от которого наследуются все типы узлов (элементы, текст, комментарии).
Узлы включают методы для взаимодействия, такие как получение родительских или дочерних узлов, клонирование и доступ к текстовому содержимому.
У элементов (Element) есть дополнительные свойства и методы, такие как работа с классами, внутренним HTML и т. д.
Детализация классов элементов:
Элементы дополнительно классифицируются по типам, как, например, HTMLInputElement для полей ввода, у которых есть уникальные свойства, например value.
Этот подход обеспечивает точное взаимодействие с разными типами HTML-элементов, соответствующих различным тегам.
Обработка событий:
В основе системы событий в DOM лежит класс EventTarget, позволяющий узлам реагировать на события, такие как клики по кнопкам.
Еще одной важной экземпляром, наследующим от EventTarget, является объект window, к которому также можно применять обработчики событий.
