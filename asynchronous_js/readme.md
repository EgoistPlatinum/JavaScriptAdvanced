Введение
Переходим к теме асинхронного программирования в JavaScript.
Изучим Event Loop, промисы и получение данных с сервера.
Синхронный vs Асинхронный код
Синхронный код: Задачи выполняются последовательно. Если одна задача занимает много времени, всё останавливается до её завершения.
Асинхронный код: Использует отложенные операции, позволяя коду продолжить выполнение без блокировки. Пример: использование setTimeout.
Однопоточность JavaScript
JavaScript работает в одном потоке, но может обрабатывать асинхронные операции с помощью Web API и Event Loop.
Возможность использования веб-воркеров и тредов на Node.js для создания дополнительных потоков.
Терминология
Конкурентность: Выполнение нескольких задач за один промежуток времени в одном потоке.
Параллельность: Специфический случай конкурентности, где задачи выполняются одновременно в разных потоках.
Многопоточность: Реализация конкурентности с использованием нескольких потоков, которые могут быть как параллельны, так и конкурентны.
Асинхронность: Делегирование выполнения операций сторонней системе (например, Web API), позволяя основному потоку продолжить выполнение.
Асинхронные операции в JavaScript
Асинхронные операции позволяют делегировать выполнение длительных задач (например, таймеры и запросы на сервер) веб-АПИ, не блокируя главный поток.
Event Loop обрабатывает выполнение отложенных задач, возвращая контроль в основной поток, когда они завершены.
